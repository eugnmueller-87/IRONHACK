Model Context Protocol (MCP) — Overview
========================================

What is MCP?
MCP is an open standard created by Anthropic that defines how AI models connect to
external data sources and tools. Think of it as a USB-C port for AI: one universal
connector that works across different models and systems.

Core Concepts
-------------
- Server: Exposes tools and resources (e.g., filesystem, database, API wrapper)
- Client: The AI application that connects to one or more servers
- Tools: Functions the model can call to take actions
- Resources: Read-only data the model can consume as context
- Prompts: Reusable prompt templates registered by the server

Transport Layers
----------------
- stdio: Local process communication (fast, simple, for local servers)
- SSE (Server-Sent Events): HTTP-based, for remote or networked servers

Why MCP?
--------
Before MCP every integration was custom: each AI app wrote its own connectors for
Slack, GitHub, Notion, etc. MCP standardises this so a single MCP server can be
reused across Claude, GPT, Gemini, or any LangChain agent.

Popular MCP Servers
-------------------
- @modelcontextprotocol/server-filesystem  — local file read/write
- @modelcontextprotocol/server-github      — GitHub issues, PRs, repos
- @modelcontextprotocol/server-postgres    — SQL queries against Postgres
- @modelcontextprotocol/server-brave-search — web search via Brave API
