LangChain Agents — Quick Reference
====================================

What is a LangChain Agent?
An agent uses an LLM as a reasoning engine to decide which tools to call and in
what order, looping until it reaches a final answer.

Key Components
--------------
- LLM / Chat Model: The brain (e.g., ChatOpenAI, ChatAnthropic)
- Tools: Functions the agent can invoke (search, calculator, MCP tools, etc.)
- Prompt: System message that defines agent behaviour and available tools
- AgentExecutor: The runtime loop that alternates between LLM calls and tool calls

Agent Types (LangChain)
-----------------------
- create_tool_calling_agent  — recommended; uses native tool-calling API
- create_react_agent         — ReAct pattern (Reason + Act), text-based
- create_openai_functions_agent — older OpenAI functions API (deprecated)

Tool-Calling Agent Example
--------------------------
from langchain import hub
from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini")
prompt = hub.pull("hwchase17/openai-tools-agent")
agent = create_tool_calling_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
result = executor.invoke({"input": "What files are in the documents folder?"})

MCP + LangChain
---------------
langchain-mcp-adapters bridges MCP servers and LangChain:
- MultiServerMCPClient manages connections to one or more MCP servers
- get_tools() returns LangChain-compatible BaseTool objects
- No manual wrapping needed
