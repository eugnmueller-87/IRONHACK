## Lab Summary

The main design choice was to keep the vector store entirely in-memory using NumPy arrays rather than introducing an external database like Pinecone or FAISS — this keeps the pipeline self-contained and makes every step transparent: chunking produces a list of strings, embedding turns them into a float matrix, and retrieval is a single matrix multiplication followed by an argsort. I used `text-embedding-3-small` for cost efficiency (it performs comparably to the large model on most retrieval tasks) and `gpt-4o-mini` for generation to stay within free-tier limits while still producing coherent answers. Chunking was done by word count with a small overlap (15 words) to avoid cutting context at paragraph boundaries. The system prompt explicitly constrains the model to answer from retrieved context only and to cite sources, which directly addresses the hallucination problem that RAG is designed to solve.
