# Lab Summary — n8n Node Analysis

The most useful nodes in practice are **HTTP Request**, **Code**, **AI Agent**, and **Set** — HTTP Request handles the majority of real-world integrations since almost every external service has a REST API; Code covers any transformation that standard nodes cannot express; AI Agent orchestrates LLM-powered reasoning loops; and Set is the everyday workhorse for reshaping data between steps. When picking a node for a task, I start from the data shape I have and the shape I need: if the gap is a simple field rename or filter, Set or IF is enough; if it requires calling an external service, HTTP Request or a purpose-built integration node (Google Sheets, OpenAI) is the right call; if it requires conditional logic that fans out to multiple paths, Switch beats chaining multiple IF nodes. My top debugging tip is to run the workflow up to the problematic node, click the node in the execution view, and compare the actual input JSON against what the node's expressions expect — most bugs are a field name mismatch or a missing `{{ }}` wrapper in an expression, and the execution log surfaces both instantly without any guesswork.
