# Lab Summary — NormalObjects Strict Complaint Processor (LangGraph)

LangGraph enforces a fixed execution order — every complaint must pass through intake, validate, investigate, resolve, and close in sequence, and the graph structure physically prevents skipping steps. This is the core difference from the LangChain agent in Lab 1 (Week 3), where the LLM chose which tools to call and in what order, producing creative but non-deterministic results. LangGraph is the right choice when you need auditable, compliance-level workflows — for example, when a failed validation must always produce a rejection record, or when an investigation must complete before a resolution can be proposed. LangChain agents are better suited to open-ended, exploratory tasks where you want the model to reason freely about which tools are relevant; the trade-off is that the agent may skip steps, repeat itself, or take unexpected paths that are hard to audit.
