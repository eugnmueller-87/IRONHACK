# Lab Summary

The workflow is a 7-node n8n pipeline triggered by a webhook POST request containing an arXiv URL: a Python code node extracts the paper ID, an HTTP Request node fetches the raw Atom XML from the arXiv API, a second Python node parses out the title, authors, abstract, and category using regex, the Anthropic Claude Haiku node summarizes the abstract into 2-3 plain-English sentences, a Notion node creates a new database page with all fields populated, and a final Respond to Webhook node returns a JSON success confirmation. The main challenge was that n8n's Python code nodes use `_items` instead of the documented `_input` variable, and after introducing the Anthropic node, all upstream fields (title, authors, arXiv URL) had to be referenced back to the Code in Python1 node using `$('Code in Python1').first().json.fieldname` because `$json` in the Notion node only sees the Anthropic response. One improvement would be to add an error-handling branch that catches arXiv rate limit errors and retries after a short delay, rather than failing silently.
