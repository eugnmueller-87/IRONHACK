# Lab Summary — Autonomous Agent Challenge Lab

The hardest part of planning this procurement agent was not the technology selection but defining the right human-in-the-loop boundaries: deciding exactly where the agent should stop and hand off to a human required understanding real procurement workflows deeply enough to know that a category manager losing control of supplier selection — even to a well-reasoned AI recommendation — would kill adoption before the project even launched. If I were to do this differently, I would start the scoping conversation by mapping the current manual process step by step before jumping to technology, because several important constraints (the P2P tool as entry point, the ERP as source of truth, the CM as mandatory gatekeeper for all RFQ/RFP initiation) only emerged through back-and-forth discussion and would have been missed with a top-down design approach. The biggest open question remains how to handle the supplier compliance gap scenario gracefully: when a promising supplier is missing a DPA or MSA, the agent needs to either block engagement or trigger a parallel contract request workflow — and the logic for when to block versus when to conditionally allow with CM override is a business decision that needs explicit sign-off from legal and procurement leadership before it can be encoded into the agent's state machine.
